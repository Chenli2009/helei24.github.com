      阴阳历算法实现(1) -- 阳历转阴历

      Post on Jan 25 2015

      阳历转阴历

      实现背景

      本人一年前实习时候参与过一个API项目,其中就包括实现阴阳历互换API接口,调研过一番其实现算法.时间久了算法详细过程大概忘记了,最近突然就想重新实现一次,于是又开始了.
      跟什么进制转换,计算器等等算法相比,阳历(即公历,就是生活中走得较快的那个全世界都在使用的日期)与阴历(即农历,旧历,就是生活中走得较慢的只有咱们中国在用的那个日期)之间的相互转换就没有一套固定的转换规律了. 对于阳历而言,其相对还算简单,大小月以及平闰年分布有规律,很方便计算,但是对于农历就相对复杂了,农历也有大小月之分,但是无平闰年之分,尤其特殊的是闰月出现的不规律,这就为算法增添了难度.但是,还是有办法实现的的!!

      算法设计

      由于阴阳历日期之间间隔时间的不规律性,每一年阴阳历日期之间的相差的天数都不一样,不能简单的对给定日期进行加减运算.其实最核心的就是得出所求年正月初一到所求日期之间的总天数就行了．所以,要求出给定阳历日期与所求阴历日期相差的天数,需要
      第一,需要确定所求阴历年份是与阳历年份相同还是少一年;
      第二,找到一个中间日期作为阴阳历转换的基准,显然,春节大年初一最为适合,确定当年大年初一的阳历时间;
      第三,确定所求年农历日期距离春节(大年初一)的具体天数;
      第四,计算阴历月日.
      显然,四步当中,第二步最为艰难,怎样确定大年初一的阳历时间呢?没有能确定的规则,只能将每年正月初一的阳历日期预先存储起来
      接下来,就为了确定农历日期,便需要获取当年农历每月大小月分布情况,闰月等等,我们仍旧预先存储.从空间和常用角度考虑,我们只需用十六进制存储,而且存储1901-2099之间两百年就够了.农历日期结构设计如下:

      算法实现

      # 返回阳历年份的总天数
def get_syear_total_days(syear):
    if (syear % 4 == 0 and syear % 100 != 0) or (syear % 400 == 0):
        return 366  #闰年
    else:
        return 365  #平年

# 返回阳历年所给月份的总天数
def get_syear_total_month_days(syear, smonth):
    days = MONTH_DAYS[smonth]    # 每月总天数
    if (syear % 4 == 0 and syear % 100 != 0) or (syear % 400 == 0):
        if smonth == 2:          # 闰年2月总天数为29天,平年为28天
            days += 1
    return days    

# 返回给定年1月1日到给定日期的总天数
def get_days_of_syear(syear, smonth, sday):
    days = sday
    for m in range(1, smonth):
        days += get_syear_total_month_days(syear, m)    
    return days

# 返回所求阴历年份, 返回正月初一到所给阳历日期(对应的阴历日期)的总阴历天数
def get_lunar_days_to_sday(syear, smonth, sday):
    lunar_days = 0    # 总阴历天数
    lyear = syear     # 阴历年份
    #假设当前阳历年份和阴历年份在同一年,然后求出当前年正月初一所在日期
    spring_month = (LUNAR_CALENDAR_TABLE[syear - 1901] & 0x60) >> 5    #春节(正月)所在阳历月份
    spring_day = LUNAR_CALENDAR_TABLE[syear - 1901] & 0x1F    #春节(正月初一)所在阳历日期              
    # 如果春节所在阳历日期在给定阳历日期之前,意味着阳历日期已翻年,阴历年份减一
    if spring_month > smonth or (spring_month == smonth and spring_day >= sday):
        lyear -= 1    #阴历年份减一
        #阳历年份和阴历年份不在同一年,阳历已翻年,所以需要重新获取正确(阴历)年份正月初一日期
        spring_month = (LUNAR_CALENDAR_TABLE[syear - 1901 -1] & 0x60) >> 5    #正确阴历年春节(正月)所在阳历月份
        spring_day = spring_day = LUNAR_CALENDAR_TABLE[syear - 1901 -1] & 0x1F    #正确阴历年春节(正月)初一所在阳历日期
        # 给定日期在给定年份的总天数 + 上年(翻年前)全年总天数 - 上年(翻年前)正月初一总天数
        lunar_days = get_days_of_syear(syear, smonth, sday) + get_syear_total_days(lyear) - get_days_of_syear(lyear, spring_month, spring_day)    #  
    # 当前阳历年份和阴历年份在同一年,即未翻年
    else:
        # 给定日期在给定年份的总天数 - 给定年份正月初一总天数
        lunar_days = get_days_of_syear(lyear, smonth, sday) - get_days_of_syear(lyear, spring_month, spring_day)

    return (lyear, lunar_days)    # 返回年份,日期总天数


# 根据给定阳历年,月,日返回阴历日期
def get_lunar_date(syear, smonth, sday):
    if syear < 1901 or syear > 2099:    # 如果给定年不在1901-2099之间, 罢工
        return   

    lyear, lunar_days = get_lunar_days_to_sday(syear, smonth, sday)    # 所求阴历年, 阴历总天数
    leap_month = (LUNAR_CALENDAR_TABLE[lyear-1901] >> 20) & 0xF  # 闰月月份,返回0表示无闰月

    lmonth = lday = 1    # 初始化农历月，日
    start_day_of_month = 1    # 用以累加, 获取农历月份和日期
    bits = 19    # 遍历获取大小月索引, 大月:1, 小月:0
    ＃ 循环所求年农历月份
    for lmonth in range(1, 14):
        l_big_month = (LUNAR_CALENDAR_TABLE[lyear-1901] >> bits) & 0x1    # 当前月大小

        if start_day_of_month + 29 + l_big_month < lunar_days:
            lmonth += 1    # 月份+1
            start_day_of_month += 29 + l_big_month    ＃ start_day_of_month＋当前农历月份总天数
        else:
            lday = lunar_days - start_day_of_month    # 得到所求日期
            break    # 结束循环 
        bits -= 1    ＃ 跳至下个月　　　　

    # 考虑闰月
    if leap_month and leap_month <= lmonth - 1:
        lmonth -= 1 #　如果闰月，由于lmonth+=1，所以需要-=１
    is_leap_month = True if lmonth == leap_month else False    ＃闰月：返回True, 非闰月：返回False    

    lunar_date_dict = {"lyear":lyear, "lmonth":lmonth, "lday":lday, "is_leap_month":is_leap_month}
    lunar_date_dict.update(get_lunar_chinese_date(lyear, lmonth, lday, is_leap_month))

    return lunar_date_dict

def get_lunar_chinese_date(lyear, lmonth, lday, is_leap_month):
    cn_lyear = CN_LUNAR_TG[(lyear - 4) % 10] + CN_LUNAR_DZ[(lyear - 4) % 12]
    cn_animal_lyear = CN_LUNAR_ANIMAR[(lyear - 4) % 12]
    cn_lmonth = CN_LUNAR_MONTH[lmonth]
    cn_lday = CN_LUNAR_DAY[lday]

    return {"cn_lyear":cn_lyear, "cn_animal_lyear":cn_animal_lyear, "cn_lmonth":cn_lmonth, "cn_lday":cn_lday}
